// Файл описывает механизм работы с версиями профилей субсидий на Портале.
// Используется внутри файлов: subsidy.adoc и subsidy_oper.adoc

Данные о новой субсидии, или новой версии субсидии поступают из реестра «Реестр субсидий ЮЛ, ИП, ФЛ (федеральные)»/«Реестр субсидий ЮЛ, ИП, ФЛ (региональные и муниципальные)» ГИИС ЭБ. 

На основе поступивших данных на Портале автоматически формируется проект профиля субсидии с признаками: «Версия 1», статус — {st_draf}. 
Профиль содержит только те данные, которые изначально были утверждены в составе Реестра субсидий ЮЛ, ИП, ФЛ  в ГИИС ЭБ. 
Для публикации профиля субсидии на Портале необходимо дополнить его информацией, согласовать и утвердить. 
Публикация профиля субсидии на открытой части Портала осуществляется автоматически после его утверждения уполномоченным лицом. 

Новая версия субсидии всегда формируется в статусе {st_draf}. 

Возможные статусы профиля субсидии: 

* {status_s_1}; 
* {status_s_2}. 

Статусы у версии профиля субсидии могут быть: 

* {status_v_s_1}. 
* {status_v_s_2}. 
* {status_v_s_3}. 
* {status_v_s_4}. 
* {status_v_s_5}. 
* {status_v_s_6}. 

Если на Портал поступает новая версия субсидии из Реестра субсидий Подсистемы, все ранее созданные версии в статусах: {st_draf}, {st_in_review}, {st_approved}, {st_rejected} переходят в статус «в архив».

В реестре {subsidy} представлен статус версии профиля.

Профиль субсидии на Портале автоматически формируется в статусе {st_draf}, и версия для редактирования тоже в статусе {st_draf}. 
Далее все действия производятся только с версией: редактирование и согласование. 
До момента, пока версия не перешла в статус {st_published}, профиль субсидии остается в статусе {st_draf}. 

Когда версия переходит в статус {st_published}, профиль субсидии автоматически переходит в статус {st_published}.
Все новые версии поступают в статусе {st_draf}.
