// Настройка групп полей, инфа про настройку каждого поля Относится к Настройке отбора, часть из настроек шаблона заявки

В конце каждой секции при необходимости добавления новой группы полей можно нажать кнопку «+ Добавить группу полей», как на <<id_image-g_os_sgf_1, Рисунке {counter:figure-number}>>.

[[id_image-g_os_sgf_1]]
.Рисунок {counter:figure-number}. Кнопка «+Добавить группу полей»
image::g_os_sgf_1.png[{image-size1}, align="center"]

После нажатия появится новая группа полей. 
Эта группа полей является редактируемой кастомной. 
При наведении курсора мышки на группу полей появятся кнопки, как на <<id_image-g_os_sgf_2, Рисунке {counter:figure-number}>>:

* image:{iconsdir}plus.png[width=16,height=16] (1) – при нажатии появляется список из видов полей (2), выбор полей которыми необходимо наполнить группу;
* image:{iconsdir}settings.png[width=16,height=16] (3) – при нажатии открывается боковое окно настроек группы;
* image:{iconsdir}delete.png[width=16,height=16] (4) – при нажатии открывается модальное окно подтверждения действий, после подтверждения группа удаляется без возможности восстановления.

[[id_image-g_os_sgf_2]]
.Рисунок {counter:figure-number}. Меню действий, и список видов полей
image::g_os_sgf_2.png[{image-size1}, align="center"]

Поля, которые можно добавить в группу и их особенности описаны ниже.

«Текстовая строка» – поле, в которое можно вручную ввести текст в одну строку, дополнительно к ранее описанным настройкам можно выставить, как на <<id_image-g_os_sgf_3, Рисунке {counter:figure-number}>>:

* пропорции поля в соотношении к группе кратное 12 (1);
* включить отображение нижнего лейбла (подсказка под полем) (2);
* зависимость от чек-бокса (3) – определяет отображение поля в зависимости от состояния чек-бокса в группе: если чек-бокс активен, то поле отображается (Прямая), если неактивен – поле скрыто (Обратная). 
Если в группе нет чек-бокса, эта настройка не применяется;
* задать ограничение по символам, при заполнении поля участником будет происходить автоматическая проверка на количество символов (4);
* настроить маску, при заполнении поля участником будет происходить автоматическая проверка по символам и их количеству, для настроек ознакомьтесь с подсказкой (иконка image:{iconsdir}help.png[width=16,height=16]) (5).

При выставлении пропорций поля в соотношении к группе прописывается цифра кратная 12. 
Например: при вводе «12» — строка будет занимать в заявке всю ширину страницы (1), при вводе значения «4» — строка  будет занимать одну треть ширины страницы заявки (2), при вводе «6» — соответственно, половину страницы (3), при вводе 3 – одну четвертую, как на <<id_image-g_os_sgf_4, Рисунке {counter:figure-number}>>.

«Текстовое поле» – поле, в которое можно вручную ввести текст в несколько строк, настройки идентичны «Текстовой строке».

«Выпадающий список» – поле, в котором участник может сделать выбор из настроенного справочника, в настройках можно составить справочник. 
Для этого необходимо в боковом окне настроек в разделе «Словарь» (Рисунок 85) нажать кнопку «Добавить» (1) и в появившемся поле (2) вписать первое название элемента справочника, после вновь нажать кнопку «Добавить» и дать название следующему элементу и т. д., как на <<id_image-g_os_sgf_5, Рисунке {counter:figure-number}>>.

«Чек–бокс» – элемент, позволяющий пользователю управлять параметром с двумя состояниями — включено и отключено, можно добавлять, как отдельное поле, где пользователь нажатием активирует флажок или снимает его. 
Можно установить один из трех типов чек–бокса, как на <<id_image-g_os_sgf_6, Рисунке {counter:figure-number}>>:

[[id_image-g_os_sgf_6]]
.Рисунок {counter:figure-number}. Типы использования чек–бокса

* в строке – при активации чек–бокса пользователю будет открываться скрытое поле для дальнейших действий (ввода данных или выбора значений), поле добавляется в ту же группу, что и чек–бокс (1);
* в группе – при активации чек–бокса пользователю будет открываться скрытая группа полей для дальнейших действий (ввода данных или выбора значений), поля добавляются в ту же группу, что и чек–бокс (2);
* обыкновенный – чек–бокс будет находиться в отдельной строке (3), как на <<id_image-g_os_sgf_7, Рисунке {counter:figure-number}>>.

«Дата» – поле, в которое можно вручную ввести дату, или посредством кнопки календарь выбрать дату из «Календаря».

«Деньги» – поле, в которое можно вручную ввести только цифры, допускается через запятую.

«Выпадающий мультисписок» – идентичен «Выпадающему списку», но позволяет пользователю выбрать несколько значений из справочника.

«Электронная почта» – поле, в которое можно ввести данные электронной почты, при заполнении будет проверка на символ @.

«Телефон» – поле, в которое можно ввести только цифры, при заполнении будет проверка на количество символ.

«Файл» – поле, в которое можно загрузить файл через диалоговое окно, или методом перетаскивания, при настройках поля можно задать тип или типы допустимых файлов, максимальное количество, которое сможет добавить пользователь, и размеры файлов, как на <<id_image-g_os_sgf_8, Рисунке {counter:figure-number}>>.

====
[%unbreakable, grid=none, frame=none, cols="1,15"]
|===
^.^| image:{iconsdir}warning.png[16,16] |
При настройках поля «Файл» необходимо указать «Разрешенные типы файла» (файлы выбираются из предложенного списка, множественным выбором) и «Максимально допустимый размер файла». 
Рекомендуем выставлять: «Размер файла»: не более 50 Мбайт, «Расширение файла»: pdf, doc, docx, tiff, jpg, jpeg, png, rtf, xls, xlsx, avi, mpg. 

Заполнение этих данных обязательно, без этих данных поле не будет отображаться в заявке при заполнении.
|===
====

«Адрес» – поле, в которое вводится адрес.
«Веб-ссылка» – поле, в которое вводится URL–ссылка, и слева от этого поля будет кнопка для возможности перейти по введённому адресу.

При необходимости настроить группу добавленных полей в боковом окне настроек группы, дополнительно к полям описанным выше имеет ряд переключателей, отвечающих за оформление группы, как на <<id_image-g_os_sgf_9, Рисунке {counter:figure-number}>>:

. «Показать звездочку–маркер наличия обязательных полей» (1), необходимо включить если группа обязательная к заполнению, рекомендуется ее включать для группы обязательных полей, чтобы пользователи могли ориентироваться при заполнении заявки. Носит информационный характер, не влияет на процент заполненности секции.
. «Имя кнопки добавления блоков» (2), необходимо прописать, если пользователь самостоятельно должен добавить еще один такой же блок для заполнения посредством этой кнопки.
. «Максимальное число блоков инпутов» (3) – можно внести ограничение при использовании кнопки «Добавления блоков».
. «Скрыть кнопку добавления» (4), необходимо включить, если кнопка добавления полей пользователем не предусмотрена.
. «Скрывать границу на группах элементов» (5), визуальное оформление блока, поля группы не будут заключены в общую рамку.
. «Скрывать название для группы» (6), если переключатель активен, то название группы полей не будет отображаться в заявке у участника.
. «Название элемента при печати» (7) можно задать новое название группы полей исключительно для печати.
. «Скрывать номер» (8), при активации порядковый номер поля будет скрыт в заявке.
. «Не выводить на печать» (9), при активации группа полей не будет отображаться в печатной форме.
