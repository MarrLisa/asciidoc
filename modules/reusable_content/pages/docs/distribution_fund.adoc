Также в виджете {v_distr_fund} отображается счетчик «К распределению» который показывает объем средств, еще доступный для распределения по остальным заявкам. 
Значение этого счетчика рассчитывается по формуле: _[Принимаемые бюджетные обязательства в отборе, указанные в настройках отбора] – [Распределено]_.

Действия, которые обновляют эти счетчики:

. Распределение денег по заявке – увеличивает значение «Распределено».
. Отключение чек-бокса {allocate} – уменьшает значение «Распределено», если по этой заявке ранее были распределены деньги.
. Снятие заявки на этапе распределения – заявка пропадает из реестра, и, если по ней ранее были распределены деньги, они высвобождаются, уменьшая значение «Распределено».

Для того, чтобы увидеть все суммы по годам и КБК необходимо воспользоваться кнопкой «Подробнее», как на <<id_image-g_df_6, Рисунке {counter:figure-number}>>.

* «КБК» – код КБК, по которому были определены принимаемые бюджетные обязательства (бюджет данного отбора).
* «Принимаемые бюджетные обязательства» – отображается сумма, указанная по данному КБК, по этому году при формировании отбора в {otbor_section} в подразделе {fund_distr} в блоке «Принимаемые бюджетные обязательства по КБК».
* «Объем распределенных средств» – отображается общая сумма распределенных средств по всем заявкам (прошедшим распределение средств) по данному КБК, по этому году при распределении средств в модальном окне «Распределение средств по КБК».
* «Остаток по отбору» – эта разница между суммой «Принимаемые бюджетные обязательства» и «Объем распределенных средств» в рамках года, допустимо, что остаток по КБК может быть меньше ноля (отрицательным).

Для последующего исключения отрицательных значений и выстраивания корректного остатка к распределению в реестре {subsidy} необходимо осуществить следующее:

* Если по соответствующему КБК в реестре {subsidy} имеется сумма остатка к распределению достаточная для исключения отрицательного значения в реестре {r_distr_fund}, то необходимо внести изменение в объявление об отборе в {otbor_section} в подразделе {fund_distr} в блоке «Принимаемые бюджетные обязательства по КБК», указав корректные суммы по КБК для исключения отрицательных значений.
* Если по соответствующему КБК в реестре {subsidy} отсутствует сумма остатка к распределению достаточная для исключения отрицательного значения в реестре {r_distr_fund}, то необходимо (1) до формирования и утверждения протокола подведения итогов осуществить в Подсистеме изменение сумм лимитов бюджетных обязательств по соответствующим КБК, имеющим отрицательное значение (добавление нового КБК (при необходимости), (2) после обновления сумм в реестре {subsidy} внести изменение в объявление об отборе в {otbor_section} в подразделе {fund_distr} в блоке «Принимаемые бюджетные обязательства по КБК», указав корректные суммы по КБК для исключения отрицательных значений.
* Если до подписания протокола подведения итогов отбора внесение изменений в суммы ЛБО по КБК в Подсистеме не представляется возможным, то допустимо утвердить протокол подведения итогов отбора с наличием в реестре {r_distr_fund} отрицательных значений по КБК. 
Отрицательные значения исчезнут, но только по мере заключения соглашений с победителями отбора и изменения в Подсистеме сумм в колонке «БО» вкладки «Финансовое обеспечение» записи субсидии в «Сведений о субсидии» («Бюджетная роспись и ЛБО»), которые будут автоматически переданы на Портал и отразятся по соответствующей субсидии в реестре {subsidy}. 
Но обращаем внимание, что данный вариант действий не позволит объявить очередной отбор получателей субсидии с использованием КБК, имеющего отрицательное значение в столбце «Остаток к распределению».

=== Отбор с проведением экспертизы заявок (двухэтапный)

:stage2: Есть второй этап

Отборы получателей субсидий бывают двух типов, второй тип – двухэтапный или отбор с оценкой, это значит, что отбор проходит два этапа, сперва рассмотрение заявок на предмет соответствия требованиям, после чего проводится экспертиза заявок.

В разделе {r_distr_fund} реестр со списком заявок в отборе с оценкой состоит из следующих колонок:

include::r_distr_fund.adoc[tags=reiting_zayav_otb_exp;all_zayav]

Колонка «Экспертиза», может иметь разный вид, в зависимости от настроек, как на <<id_image-g_df_20, Рисунке {counter:figure-number}>>.

[[id_image-g_df_20]]
.Рисунок {counter:figure-number}. Колонка «Экспертиза», в отборе, где не предусмотрено редактирование среднего балла
image::g_df_20.png[{image-size1}, align="center"]
 
Если при настройке и публикации отбора в настройках отбора в {application_section} в подразделе {application_exp} 
(1) была включена настройка редактирования среднего балла (2), как на <<id_image-g_df_21, Рисунке {counter:figure-number}>> (возможность редактирования балла используется в случае, если экспертный балл носит рекомендательный характер согласно правилам отбора), то в реестре также отображаются заявки с отложенным статусом {notsupported}.
Это нужно для того, чтобы была возможность добавить баллы неподдержанной заявке.

[[id_image-g_df_21]]
.Рисунок {counter:figure-number}. В настройках отбора включена настройка редактирования среднего балла
image::g_df_21.png[{image-size1}, align="center"]

При включенной настройке редактирования среднего балла, в реестре в колонке «Экспертиза» будут отображаться две дополнительные колонки, как на <<id_image-g_df_22, Рисунке {counter:figure-number}>>:

. «Измененный средний балл» (1) – отображает средний балл (2) (изначально значение равно значению из колонки «Средний балл», после внесения изменений отображается измененное значение) и кнопку для перехода к редактированию среднего балла (3).
. «Автоматический комментарий» (4) – содержит автоматически генерируемые комментарии, поясняющие, по каким критериям оценки были повышены или понижены баллы (отображается только в заявках, где было изменено значение среднего балла).
. «Причина изменения среднего балла» (5) – здесь отображается комментарий внесенный в модальное окно «Средний балл по критериям» с указанной причиной изменения среднего балла.

[[id_image-g_df_22]]
.Рисунок {counter:figure-number}. Колонка «Экспертиза». Включена настройка редактирования среднего балла
image::g_df_22.png[{image-size1}, align="center"]
 
Для редактирования среднего балла необходимо нажать кнопку image:{iconsdir}comment.png[width=16,height=16] для редактирования среднего балла, после чего откроется модальное окно «Средний балл по критериям», как на <<id_image-g_df_23, Рисунке {counter:figure-number}>>, с информацией номер заявки и название заявителя (1) и таблицей с критериями, состоящей из следующих колонок:

* «Нумерация критериев» (2) – порядковый номер критерия;
* «Название критерия» (3) – название критерия; 
* «Средний балл» (4) – отображается средний балл, который считается автоматически по результатам экспертизы;
* «Изменение среднего балла» (5) – представлено поле для внесения изменения среднего балла для каждого критерия. Можно повысить или понизить балл, нажав соответствующие кнопки (6), или ввести значение вручную. Значение изменения может быть как положительным, так и отрицательным.
* «Измененный средний балл» (7) – отражает итоговый средний балл с учетом изменений по каждому критерию. Это поле автоматически проверяется, если после пересчета коэффициента среднего балла по критерию формируется отрицательное значение (8), сохранение изменений блокируется (9).

[[id_image-g_df_23]]
.Рисунок {counter:figure-number}. Модальное окно «Средний балл по критериям»
image::g_df_23.png[{image-size2}, align="center"]

* «Причина изменения среднего балла» – текстовое поле для внесения комментария по причине изменения среднего балла, как на <<id_image-g_df_24, Рисунке {counter:figure-number}>>. 
Обязательно для заполнения. 
Поле отображается только при не нулевом значении коэффициента изменения хотя бы одного критерия по заявке.

[[id_image-g_df_24]]
.Рисунок {counter:figure-number}. Поле «Причина изменения среднего балла»
image::g_df_24.png[{image-size1}, align="center"]
 
После внесения изменений среднего балла по критерию (–ям) необходимо нажать кнопку «Подтвердить», после чего в реестре изменятся данные и активируется кнопка «Пересчитать рейтинг».
