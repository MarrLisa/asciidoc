=== Рассмотрение поступивших заявок

Изначально требования устанавливаются Организатором отбора при настройке и публикации отбора. 
Указанные требования опубликованы в разделе «Рассмотрение заявок» Объявления об отборе, для просмотра всего списка требований необходимо нажать кнопку «Показать все» (1) , как на <<id_image-g_ra_cz_1, Рисунке {counter:figure-number}>>.

[[id_image-g_ra_cz_1]]
.Рисунок {counter:figure-number}. Требования к участнику отбора
image::g_ra_cz_1.png[{image-size}, align="center"] 
 
Для рассмотрения поступившей заявки на предмет соответствия требованиям необходимо в реестре {list_applications_cz} выбрать назначенную заявку. 
Для перехода на страницу заявки необходимо нажать на регистрационный номер заявки, как на <<id_image-g_ra_cz_2, Рисунке {counter:figure-number}>>.

После перехода открывается страница заявки.
Страница организована следующим образом, как на <<id_image-g_ra_cz_3, Рисунке {counter:figure-number}>>:

. Информационная строка (1) — единая для всех заявок, содержит номер заявки, наименование проекта и статус.
. Боковое меню для навигации по секциям заявки (2).
. {check_results} пункт меню для валидатора (3) — рабочий блок для резюмирования проведенной проверки.
. Блок проверки заявителя, который отображает результаты автоматических проверок (4).
. Кнопка для отправки запроса для обновления данных по автоматической проверке (5).
. Блок с информацией по секции из заявки или дополнительных пунктов меню (6).
. Кнопка «к валидации» для быстрого перехода в секцию {check_results} (7).

После проверки информации в секциях заявки и прикрепленной документации необходимо перейти в секцию {check_results} в боковом меню (1) или воспользоваться «плавающей» кнопкой «к валидации» (2), как на <<id_image-g_ra_cz_4, Рисунке {counter:figure-number}>>.

На странице секции {check_results}, в строке заголовка, отображается статус валидации, который может быть:

* {not_started};
* {in_progress};
* {completed}.

Основная часть страницы – это блок «Критерии»: список требований, установленных правилами проведения конкурса (Постановление Правительства Российской Федерации от 16 ноября 2020 г. № 1842 «Об утверждении Правил проведения конкурса на заключение соглашения об оказании государственных (муниципальных) услуг в социальной сфере») и указанных при настройке отбора, и списка оснований для отклонения заявок, указанных также при настройке отбора, с кнопками image:{iconsdir}rev_yes.png[width=16,height=16] «соответствует» и image:{iconsdir}rev_no.png[width=16,height=16] «не соответствует», как на <<id_image-g_ra_cz_5, Рисунке {counter:figure-number}>>.

Некоторые требования привязаны к итогам автоматической проверки, которая происходит с момента заполнения заявки.
Виджет с автоматической проверкой содержит данные по проверке заявителя по данным государственных информационных систем. 
Количество проверок отличается от типа субъекта экономической деятельности, как на <<id_image-g_ra_cz_6, Рисунке {counter:figure-number}>>.

Итоги автоматической проверки частично привязаны к требованиям в секции {check_results}. 
В блоке учитываются автоматические проверки из списка:

* Юридическое лицо не проходит процедуру ликвидации (ЮЛ);
* Деятельность юридического лица не приостановлена (ЮЛ);
* Деятельность индивидуального предпринимателя не прекращена	(ИП);
* Отсутствует в реестре недобросовестных исполнителей	(ЮЛ, ИП, ФЛ).

Если автоматическая проверка находится в процессе (1), то это же требование в списке не будет иметь кнопок image:{iconsdir}rev_yes.png[width=16,height=16] и image:{iconsdir}rev_no.png[width=16,height=16] (2), необходимо дождаться итогов автоматической проверки или запустить повторную проверку, нажатием кнопку «Повторить» в виджете (3), как на <<id_image-g_ra_cz_7, Рисунке {counter:figure-number}>>.

[[id_image-g_ra_cz_7]]

image::g_ra_cz_7.png[{image-size}, align="center"] 

При нажатии кнопки «Повторить» происходит повторный запрос в государственные информационные системы на проверку заявителя по соответствующим требования к участнику отбора. 

Если автоматическая проверка не пройдена (1), то это же требование в списке будет отмечено image:{iconsdir}rev_no.png[width=16,height=16] (2), с автоматическим комментарием (3), содержащим текст, указанный применительно к этому требованию при формировании и утверждении Профиля субсидии.
Отметку и комментарий по требованию невозможно изменить. 
Автоматически image:{iconsdir}rev_no.png[width=16,height=16] отмечается пункт в критериях «Нет основания для отклонения», без возможности изменить. 
Данные по требованиям, не прошедшим проверку, автоматически пробрасываются в конец списка — итоги проверки (4), как на <<id_image-g_ra_cz_8, Рисунке {counter:figure-number}>>.

image::g_ra_cz_8.png[{image-size}, align="center"] 
 
Если автоматическая проверка пройдена (1), то это же требование в списке будет отмечено image:{iconsdir}rev_yes.png[width=16,height=16] (2).
Отметку по требованию невозможно изменить, как на <<id_image-g_ra_cz_9, Рисунке {counter:figure-number}>>.

Остальные требования, которые не привязаны к автоматической проверке, необходимо проверить и задать результат вручную, нажатием кнопки image:{iconsdir}rev_yes.png[width=16,height=16] (1) или image:{iconsdir}rev_no.png[width=16,height=16] (2), как на <<id_image-g_ra_cz_10, Рисунке {counter:figure-number}>>.

Если при проверке проекта и (или) заявителя было выявлено несоответствие требованиям, установленным правилами проведения конкурса, то при нажатии кнопки «не соответствует» (1) появится обязательное поле для комментария (2) от валидатора. 
Также автоматически отмечается image:{iconsdir}rev_no.png[width=16,height=16] пункт в критериях «Нет основания для отклонения заявок», без возможности изменить (3).
Критерий с перечнем оснований для отклонения заявки под названием «Нет основания для отклонения» — обязательный критерий для подведения итогов проверки заявки. 
Оценка по нему проставляется автоматически – если есть хоть один критерий, по которому стоит image:{iconsdir}rev_no.png[width=16,height=16], то у данного критерия тоже будет image:{iconsdir}rev_no.png[width=16,height=16], в открывшемся списке оснований необходимо обязательно проставить флажки в чек-боксах подкритериев (4) и добавить текст причины отклонения в поле «Комментарий» (5). 
Основание вместе с комментарием автоматически пробрасываются в итоги проверки (6), как на <<id_image-g_ra_cz_11, Рисунке {counter:figure-number}>>.

По результатам выставления отметок image:{iconsdir}rev_yes.png[width=16,height=16] и image:{iconsdir}rev_no.png[width=16,height=16] автоматически становится доступным только одно решение (Допущен/Не допущен).
Решение, которое доступно, подсвечивается ярким цветом. 

После нажатия активной кнопки появится модальное окно «Подтверждение», где необходимо в подтверждение своего решения нажать кнопку «Подтвердить», как на <<id_image-g_ra_cz_16, Рисунке {counter:figure-number}>>.

После этого заявке будет задан отложенный статус в соответствии с выбором.

Страница {check_results} скроет все критерии, под названием секции в меню будет отображаться итог рассмотрения, как на <<id_image-g_ra_cz_17, Рисунке {counter:figure-number}>>.

Заявка после окончания валидации может быть возвращена на повторное рассмотрение, у таких заявок в секции {check_results} рассмотрение отображается поле «Версия результатов». 
После выбора версии в этом поле можно посмотреть ранние версии результатов проверки. 
Для выбора версии в поле выпадающего списка «Версии результатов» надо нажать image:{iconsdir}down.png[width=16,height=16] и выбрать интересующую версию. 

После создания новой версии валидации валидатору необходимо провести повторную валидацию. 
При этом новая версия валидации может быть назначена другому валидатору.

В виджете «Версия результатов» доступны все ранее проведенные валидации, как на <<id_image-g_ra_cz_18, Рисунке {counter:figure-number}>>.
