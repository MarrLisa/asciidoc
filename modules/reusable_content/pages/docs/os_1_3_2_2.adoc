Для перехода в раздел «Шаблон заявки» необходимо на странице «Настройки шаблона» нажать на пункт в боковом меню.

После перехода в раздел «Шаблон заявки» открывается страница настроек, как на <<id_image-g_os_1_3_2_2_1, Рисунке {counter:figure-number}>>:

. Заголовок – отображается название отбора, для которого настраивается данный вид шаблона (1);
. Кнопка «Распечатать шаблон» – для выведения печатной формы данного шаблона в формате PDF (2);
. Боковое меню для возврата в разделы {general_setting} и «Дополнительные настройки» (3);
. Боковое меню с предустановленными секциями шаблона, для настройки и перехода в каждую из них (4);
. Основное окно секции для настроек полей и содержания (5).

Для настройки секций шаблона бокового меню, необходимо навести курсор мыши на соответствующую секцию. 
Появятся дополнительные кнопки настроек, с помощью которых можно, как на <<id_image-g_os_1_3_2_2_2, Рисунке {counter:figure-number}>>:

* удалить секцию – при нажатии на «–» (1) откроется модальное окно для подтверждения действия, после подтверждения секция будет скрыта;
* переместить секцию выше/ниже относительно других секций (2);
* добавить новую секцию – при нажатии на «+» в конце списка секций (3) появляется новая секция доступная для настроек;
* отобразить скрытые секции (4) – при активации переключателя «Скрытые секции», если до этого секции были скрыты они отобразятся в списке с кнопкой их восстановления видимости (5).

[[project_info]]
====== Секция {project_info}

В секции {project_info}, также, как почти во всех остальных секциях, есть возможность скопировать настройки шаблонов с предыдущих отборов.  
Скопировать настройки невозможно только в секциях: {project_plan}, {project_budget}, {project_apply} и {project_withdraw}. 

Для этого надо в верхней части секции в блоке «Загрузка существующих настроек» выбрать из списка ранее опубликованных отборов (1), тот, у которого необходимо скопировать настройки шаблона и после нажать кнопку «Применить» (2), как на <<id_image-g_os_1_3_2_2_3, Рисунке {counter:figure-number}>>.

[[id_image-g_os_1_3_2_2_3]]
.Рисунок {counter:figure-number}. Загрузка существующих настроек
image::g_os_1_3_2_2_3.png[{image-size1}, align="center"]

Также, как почти во всех остальных секциях, есть возможность оставить текст подсказки, если есть такая необходимость.
Добавить подсказку невозможно только в секциях: {participant_requirements}, {recipient_data}, {project_apply} и {project_withdraw}.

В верхней части секции для добавления подсказки необходимо активировать переключатель «Предупреждающая подсказка под заголовком секции» (1), когда переключатель активирован, появляется дополнительное поле «Текст подсказки» (2), где можно ввести соответствующий текст, как на <<id_image-g_os_1_3_2_2_4, Рисунке {counter:figure-number}>>.

В шаблонах уже сформированы предустановленные поля для заявителей. 
Поля и группы полей в шаблоне есть трех типов:

* не редактируемые — поля являются предустановленными и обязательными, их невозможно редактировать и удалять, в секции {project_info} это первые три поля, как на <<id_image-g_os_1_3_2_2_5, Рисунке {counter:figure-number}>>.

[[id_image-g_os_1_3_2_2_5]]

* редактируемые типизированные — поля являются предустановленными и обязательными, их можно редактировать, но нельзя удалить, в секции {project_info} это поля 4–7, при наведении курсора мыши на поле появляется иконка image:{iconsdir}settings.png[width=16,height=16], как на <<id_image-g_os_1_3_2_2_6, Рисунке {counter:figure-number}>>.

[[id_image-g_os_1_3_2_2_6]]
.Рисунок {counter:figure-number}. Редактируемое типизированное поле секции

* редактируемые кастомные — поля могут быть предустановленными или добавленными, эти поля не являются обязательными, их можно редактировать и удалять, в секции {project_info} это группа полей 8, при наведении курсора мыши на поле появляется меню, где есть иконка image:{iconsdir}settings.png[width=16,height=16] и иконка image:{iconsdir}delete.png[width=16,height=16], как на <<id_image-g_os_1_3_2_2_7, Рисунке {counter:figure-number}>>.

[[id_image-g_os_1_3_2_2_7]]

У полей, в которых доступен функционал редактирования настроек, для того чтобы перейти к ним необходимо навести курсор на нужное поле и нажать на иконку image:{iconsdir}settings.png[width=16,height=16].
После нажатия открывается боковое окно настроек данного поля. Основные поля настроек, как на <<id_image-g_os_1_3_2_2_8, Рисунке {counter:figure-number}>>:

* «Последовательный номер» – вручную задаётся номер поля в заявке;
* переключатель «Скрывать авто нумерацию» – при активации, будет скрыта системная нумерация, при этом позиция поля сохранится;
* «Системное имя» – выставляется автоматически;
* «Название» – вводится вручную, редактируется; 
* «Плейсхолдер» – вводится вручную, редактируется, заполняется при необходимости внутренней подсказки (текста внутри поля);
* «Подсказка» – вводится вручную, редактируется, заполняется при необходимости внешней подсказки (текста над полем, скрытым под иконкой image:{iconsdir}help.png[width=16,height=16]);
* «Режим доступа» – выбирается одно значение из списка:
** «Скрыт» – при выборе поле будет скрыто от пользователя;
** «Просмотр» – при выборе поле будет доступно для просмотра пользователя, но не редактирования, вне зависимости от статуса заявки;
** «Редактирование» – при выборе поле будет доступно для редактирования пользователя в статусах заявки, допускающих это согласно статусной модели;
** «Обязательное поле» – при выборе поле будет обязательным для заполнения, это поле будет участвовать в автопроверке на заполненность.

. В шаблонах для юридических лиц:

.. «Наличие договора о совместной деятельности».
. В шаблонах для индивидуальных предпринимателей:
.. «Наличие договора о совместной деятельности».
. В шаблонах для физических лиц:
.. «Наличие договора о совместной деятельности».

====== Секция {project_apply}

В секции {project_apply} не все поля доступны для редактирования, это обусловлено тем, что в зависимости от того для какого типа субъекта экономической деятельности шаблон, в нем предустановлены правила подачи заявки.

Поле «Провайдеры подписания» заполнено по умолчанию и не может быть изменено. 
Оно автоматически заполняется в зависимости от типа пользователей, для которых создается шаблон. 
Если шаблон предназначен для юридического лица или индивидуального предпринимателя, то в этом поле будет указана «усиленная квалифицированная электронная подпись». 
А если шаблон предназначен для физического лица, то заявка будет подписана простой электронной подписью, подтвержденной учетной записью этого физического лица на Госуслугах.

В секции {project_apply}  можно отредактировать текст модального окна «Отправка заявки», которое будет отображаться пользователю сразу после нажатия кнопки «Подать заявку» и текст инструкции.

Поле «Текст положительной проверки» текстовое поле необязательно к заполнению. 
Если оставить поле «Текст положительной проверки» незаполненным, при прохождении проверки пользователю будет выводиться сообщение по умолчанию. 
Если необходимо персонализировать это сообщение – введите необходимый текст в поле.

{but_save_clear}.
