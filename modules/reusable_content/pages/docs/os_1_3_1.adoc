В окне необходимо заполнить поля:

* «Текст требования для валидатора» – поле заполняется вручную, этот текст будет видеть валидатор во время проверки заявки. Система автоматически проверяет текст на дублирование с ранее созданными требованиями.
* «Краткое название требования для пользователя» – поле заполняется вручную, этот текст будет отображаться на публичной странице объявления об отборе, как заголовок требования, а также его будет видеть участник после этапа валидации, если по этому требованию его заявка не пройдет проверку.
* «Текст требования для пользователя» – поле заполняется вручную, этот текст будет отображаться на публичной странице объявления об отборе, как основной текст требования, а также его будет видеть участник после этапа валидации, если по этому требованию его заявка не пройдет проверку.
* «Тип субъекта экономической деятельности» – поле заполняется выбором одного или нескольких значений из справочника. 
От выбранного значения будет зависеть применение этого требования к соответствующему типу заявителя (например, если указано «Юридическое лицо», то к индивидуальном предпринимателю и физическому лицу данное требование применяться не будет).
* «Тип подтверждения участником отбора» – поле заполняется выбором одного из значений из справочника. 
Выбор определенного значения влияет на то, какой элемент будет предоставлен заявителю для подтверждения требования. 
Это может быть текстовое поле для ввода информации, файловое поле для загрузки документа или чек-бокс.
