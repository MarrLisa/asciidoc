=== {protocol_summary}

==== Формирование проекта протокола подведения итогов

Формирование протокола подведения итогов отбора исполнителей услуг по социальному заказу доступно при наступлении событий:

* все заявки, поданные на отбор, в статусе {approved} прошли экспертную оценку и им назначены отложенные статусы ({supported}/{notsupported}), как на <<id_image-g_ps_cz_1.1, Рисунке {counter:figure-number}>>;
* по всем заявкам с отложенным статусом {supported} завершено распределение средств.

[[id_image-g_ps_cz_1]]
.Рисунок {counter:figure-number}. Колонка «Отложенный статус» после завершения валидации
image::g_ps_cz_1.png[{image-size}, align="center"]

В случае наступления обоих событий в разделе {list_applications_cz}, при выборе отбора в блоке «Протоколы» отобразится кнопка «Сформировать протокол подведения итогов».

Для формирования протокола необходимо в разделе {list_applications_cz} развернуть скрытый блок «Протоколы».

В появившемся поле из справочника отборов необходимо выбрать номер отбора, для которого будет формироваться протокол, для удобства можно ввести часть шифра отбора для фильтрации.

После выбора отбора в блоке «Протоколы» справа от информации по протоколу рассмотрения заявок будет кнопка «Сформировать протокол подведения итогов», как на <<id_image-g_ps_cz_2, Рисунке {counter:figure-number}>>.

[[id_image-g_ps_cz_2]]
.Рисунок {counter:figure-number}. Кнопка «Сформировать протокол подведения итогов»
image::g_ps_cz_2.png[{image-size2}, align="center"]

В зависимости от выбранного отбора в блоке протоколов в реестре {list_applications_cz} автоматически происходит фильтрация заявок.

Для формирования протокола подведения итогов необходимо нажать кнопку «Сформировать протокол подведения итогов».

В Приложении № 1 выводятся все заявки, которые имеют отложенный статус {approved} и {notsupported}, приложение содержит таблицу  «Итоговые оценки предложений, сведения о порядковом номере, присвоенном каждому предложению на участие в отборе» с колонками:

* {rating_cz}.
* {reviewed_application_number}.
* {application_date}.
* {applicant_info}.
* Итоговая оценка заявки, балл — выводится средний балл по всем критериям от всех назначенных на заявку экспертов.

Вторая таблица в Приложении №1 «Оценка предложений по критериям» содержит:

* {rating_cz}.
* {reviewed_application_number}.
* Критерий № … — выводится средний балл по каждому из критериев между разными экспертами.

Под таблицей выводится список критериев с расшифровкой: Критерий 1 - название критерия 1 установленного при настройках отбора.

Приложение №2 «Сведения о победителях отбора на заключение соглашения об оказании государственных (муниципальных) услуг в социальной сфере с распределением объема услуг» содержит таблицу с колонками:

* {rejected_application_number}.
* {applicant_info}.
* Информация об услуге — наименование услуги и распределенный объем услуги (объем, установленный в ходе распределения объема услуг).
